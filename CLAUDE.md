# CLAUDE.md — Content-Agent

## О проекте
Еженедельный автономный агент: собирает контент-тренды из бесплатных открытых источников, анализирует и присылает отчёт в Telegram. Оператор — Maria, НЕ программист: объясняй по-русски простыми словами, по шагам, сам выполняй команды в терминале где можно, разрешения спрашивай коротко.

## Цель
Без участия оператора находить: (А) залетающие форматы коротких видео, (Б) актуальные темы, (В) боли аудитории, (Г) готовые заголовки и сценарии — и раз в неделю слать отчёт в Telegram.

## Ниши
фэн-шуй, китайская метафизика, астрология, биоэнергетика, новости планетарного уровня и инновации.

## Языки запросов
RU, EN, ES, ZH, DE. Иноязычные находки переводить и сводить к русскому в отчёте.

## Жёсткие ограничения
- Бюджет $0: только бесплатные источники и бесплатные тарифы API.
- НЕ парсить Instagram и TikTok, НЕ использовать платные парсеры (Apify и т.п.).
- Короткие форматы брать с YouTube Shorts как прокси + сигналы из новостей/трендов.

## Источники данных (бесплатные)
- YouTube Data API v3 — основной (Shorts + длинные ролики, статистика, комментарии).
- GDELT — мировые новости на многих языках.
- Google News RSS — новости по запросам на 5 языках.
- Reddit API — обсуждения и боли (опционально).
- Google Trends / pytrends — растущие запросы (опционально, бывает нестабилен).

## LLM
Google Gemini API (бесплатный тариф) — основной; альтернатива Groq. Ключ из .env: GEMINI_API_KEY.

## Доставка
Telegram Bot API: краткое саммари сообщением + полный отчёт файлом-вложением (лимит сообщения 4096 символов).

## Анализ
1. Форматы: классифицировать топ-видео (хук, длина, монтаж, звук, CTA).
2. Темы: растущие темы + привязка к календарю ниш (ретрограды, затмения, китайский календарь, фестивали).
3. Боли: извлекать реальные боли/страхи/вопросы из комментариев и обсуждений.

## Метод «боль-спрос» и папка references/
- Читать ВСЕ файлы из папки references/ как библиотеку проверенных формул и примеров.
- references/headline_formulas.json — 100 формул, размечены по типам; типы "боль" и "ошибка" приоритетные.
- Логика: извлечь настоящую боль аудитории → подобрать ПО СХОДСТВУ 2–3 формулы (приоритет "боль"/"ошибка") → переписать находку в готовый заголовок, где боль названа прямо и создаётся спрос на решение → пометить такие заголовки как «боль-спрос» и выдавать их ПЕРВЫМИ.

## Сценарии (для каждой выбранной идеи — ОБА варианта)
- A. Короткое видео 15–60 сек: Хук (0–3с) → Раскрытие → Удержание → CTA + формат-референс.
- B. Длинный ролик ~20 мин: вступление-хук → 4–6 смысловых блоков → крючки между блоками → вывод → CTA.

## Скоринг и дедупликация
- Видео: velocity = просмотры / часы_с_публикации + engagement-rate.
- Дедуп: seen.json — хранить ID показанных находок, не повторять между запусками.

## Структура отчёта
Залетающие форматы → Актуальные темы → Боли аудитории → Заголовки (боль-спрос первыми) → Сценарии (короткий + 20 мин).

## Расписание / хостинг
GitHub Actions, cron раз в неделю (понедельник). Настраивается на Фазе 6. СЕЙЧАС GitHub не трогаем.

## Секреты
Ключи (TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, GEMINI_API_KEY, YOUTUBE_API_KEY) — только в .env. .env в .gitignore, никогда не коммитить и не выводить значения в ответах.

## Фазы
- Фаза 1: каркас + проверка Telegram.
- Фаза 2: коннекторы сбора (бесплатные источники).
- Фаза 3: скоринг + дедупликация.
- Фаза 4: анализ + генерация (боль-спрос, оба формата сценариев).
- Фаза 5: сборка отчёта + отправка.
- Фаза 6: GitHub Actions, автозапуск раз в неделю.
